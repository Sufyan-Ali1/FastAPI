# Neighborhood Services API

A Phase 1 FastAPI backend project for managing local service providers and reviews using JSON files as storage.

## Features

- Create service providers
- List providers with search, filters, and pagination
- Get provider details
- Update provider details
- Delete providers
- Add reviews for providers
- List reviews for a provider
- List available categories
- Get top-rated providers
- Get provider summary

## Tech Stack

- Python
- FastAPI
- Uvicorn
- Pydantic
- JSON file storage

## Project Structure

```text
neighborhood_services_api/
    main.py
    schema.py
    enums.py
    helpers.py
    requirements.txt
    README.md
    data/
        providers.json
        reviews.json
```

## Setup

Create and activate a virtual environment:

```bash
python -m venv venv
venv\Scripts\Activate.ps1
```

Install dependencies:

```bash
pip install -r requirements.txt
```

Run the API:

```bash
uvicorn main:app --reload
```

Open the docs:

```text
http://127.0.0.1:8000/docs
```

## Environment Variables

The project uses `.env` for JSON file paths:

```text
PROVIDERS=data/providers.json
REVIEWS=data/reviews.json
```

Absolute paths also work, but relative paths are easier to move between machines.

## Sample Requests

### 1. Create Provider

```http
POST /providers
Content-Type: application/json
```

```json
{
  "name": "City Home Painter",
  "email": "contact@cityhomepainter.example",
  "category": "painter",
  "city": "Lahore",
  "area": "Model Town",
  "phone": "+92-300-9001122",
  "hourly_rate": 1700,
  "years_experience": 4,
  "available": true,
  "skills": ["wall painting", "texture paint"]
}
```

### 2. List Providers

```http
GET /providers?page=1&limit=10
```

### 3. Search Providers

```http
GET /providers?search=plumbing
```

### 4. Filter By Category

```http
GET /providers?category=plumber
```

### 5. Filter By City And Availability

```http
GET /providers?city=Lahore&available=true
```

### 6. Filter By Rate Range

```http
GET /providers?min_rate=1000&max_rate=2500
```

### 7. Get Provider By ID

```http
GET /providers/{provider_id}
```

### 8. Update Provider

```http
PUT /providers/{provider_id}
Content-Type: application/json
```

```json
{
  "name": "Updated Provider Name",
  "email": "updated.provider@example.com",
  "category": "cleaner",
  "city": "Islamabad",
  "area": "F-10",
  "phone": "+92-300-5557788",
  "hourly_rate": 2000,
  "years_experience": 6,
  "available": true,
  "skills": ["deep cleaning", "office cleaning"]
}
```

### 9. Delete Provider

```http
DELETE /providers/{provider_id}
```

Expected success status:

```text
204 No Content
```

### 10. Add Review

```http
POST /providers/{provider_id}/reviews
Content-Type: application/json
```

```json
{
  "reviewer_name": "Ahmed",
  "rating": 5,
  "comment": "Excellent service and professional behavior."
}
```

### 11. List Provider Reviews

```http
GET /providers/{provider_id}/reviews
```

### 12. List Categories

```http
GET /categories
```

### 13. Get Top Rated Providers

```http
GET /providers/top-rated?rating=4
```

### 14. Get Provider Summary

```http
GET /providers/{provider_id}/summary
```

## Data Files

Provider records are stored in:

```text
data/providers.json
```

Review records are stored in:

```text
data/reviews.json
```

Data persists after server restart because the API reads from and writes to these JSON files.

## Notes

- This project intentionally uses JSON files instead of a database because databases are covered in a later phase.
- This project does not use routers, dependency injection, authentication, middleware, or response models because those are outside Phase 1 scope.
- IDs are UUID strings generated by the API.
